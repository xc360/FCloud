# 【开放】上传管理

## 上传文件

**接口地址**:`/open/upload_file`

**请求方式**:`POST`

**请求数据类型**:`multipart/form-data`

**响应数据类型**:`*/*`

**接口描述**:<p>开放接口,签名方式上传,文件储存在该磁盘的目录下，1，单次上传，传入的size和文件的size必须相等
2，分段上传，首次请求fileIndex必须等于零，根据返回的renewalSize确定每次上传多少
3，md5签名顺序appId，folderNames，randomStr，timeStamp</p>

**请求参数**:

| 参数名称       | 参数说明                                                                                                      | 请求类型  | 是否必须  | 数据类型           | schema |
|------------|-----------------------------------------------------------------------------------------------------------|-------|-------|----------------|--------|
| file       | 文件数据                                                                                                      | body  | true  | string         ||
| diskNo     | 磁盘号，必填                                                                                                    | query | false | string         ||
| fileIndex  | 文件续传开始的位置                                                                                                 | query | false | integer(int64) ||
| folderPath | 文件夹路径                                                                                                     | query | false | string         ||
| groupCode  | 组标识                                                                                                       | query | false | string         ||
| hashCode   | 文件hash值，文件的唯一值，md5加密计算                                                                                    | query | false | string         ||
| name       | 文件名称                                                                                                      | query | false | string         ||
| restCode   | rest接口标识                                                                                                  | query | false | string         ||
| resumed    | 是否开启断点续传，0:开启，1:不开启                                                                                       | query | false | string         ||
| sign       | 签名，必填                                                                                                     | query | false | string         ||
| signData   | 签名的数据，json格式，必须有timeStamp，randomStr两个字段，时间戳和随机支付串                                                         | query | false | string         ||
| signType   | 签名类型，必填md5：md5签名，使用BASE64编码数据。aes：aes签名，使用BASE64编码数据。aes_md5：aes和md5结合签名，数据使用aes加密，签名使用md5，适用于安全级别比较高的调用。 | query | false | string         ||
| size       | 文件大小                                                                                                      | query | false | integer(int64) ||
| status     | 状态：0：有效，1：无效                                                                                              | query | false | string         ||

**响应状态**:

| 状态码 | 说明           | schema    |
|-----|--------------|-----------| 
| 200 | OK           | UploadDto |
| 201 | Created      ||
| 401 | Unauthorized ||
| 403 | Forbidden    ||
| 404 | Not Found    ||

**响应参数**:

| 参数名称        | 参数说明                   | 类型             | schema         |
|-------------|------------------------|----------------|----------------| 
| downloadUrl | 下载地址                   | string         ||
| fastFinish  | 是否是秒传,true:是,false:不是  | boolean        ||
| fileIndex   | 文件续传索引                 | integer(int64) | integer(int64) |
| filePath    | 文件存储位置，网盘文件夹路径         | string         ||
| finish      | 是否完成,true:完成,false:未完成 | boolean        ||
| hashCode    | 文件hash值                | string         ||
| renewalSize | 每次上传文件大小               | integer(int64) | integer(int64) |
| uploadUrl   | 上传地址                   | string         ||

**响应示例**:

```javascript
{

    true,
```
